---
title: Quick Start
description: Set up ZenithPay and execute a first policy-gated payment
---
This guide takes you from clone to first successful payment on X Layer.

## Prerequisites
- `bun`
- `foundry` (`forge`)
- OKX API credentials (`OKX_API_KEY`, `OKX_SECRET_KEY`, `OKX_PASSPHRASE`)
- X Layer RPC access

## 1. Clone and install
```bash
git clone https://github.com/zenith-hq/zenithpay-xlayer.git
cd zenithpay-xlayer
bun install
```

## 2. Configure environment variables
Set required variables.

`api/.env`:
```bash
XLAYER_RPC_URL=https://rpc.xlayer.tech
OKX_API_KEY=...
OKX_SECRET_KEY=...
OKX_PASSPHRASE=...
SPEND_POLICY_ADDRESS=0x...
SUPABASE_URL=...
SUPABASE_ANON_KEY=...
ZENITHPAY_API_KEY_SECRET=...
```

`web/.env.local`:

```bash
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_API_URL=http://localhost:3001
```

## 3. Deploy `SpendPolicy.sol`
```bash
cd contracts
forge build
forge test
forge script script/Deploy.s.sol --rpc-url $XLAYER_RPC_URL --broadcast
```

Copy deployed address into `api/.env` as `SPEND_POLICY_ADDRESS`.

## 4. Run locally
```bash
cd web && bun dev   # http://localhost:3000
cd api && bun dev   # http://localhost:3001
```

## 5. Register agent and activate policy

Open onboarding:

```text
http://localhost:3000/onboarding
```

Connect OKX wallet, sign, and activate policy for your agent.

## 6. Execute first payment

```bash
curl -X POST http://localhost:3001/pay \
  -H "Authorization: Bearer $ZENITHPAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "agentAddress": "0xYOUR_AGENT",
    "serviceUrl": "http://localhost:3001/sell/agent-intel",
    "maxAmount": "0.01",
    "intent": "Test x402 seller flow"
  }'
```

Expected outcomes:

- `approved` with `txHash`
- `pending` with `approvalId` if above threshold
- `blocked` with reason if policy hard-limit fails

Continue with:

- `Authentication` for integration modes
- `REST API` for endpoint details
- `MCP Server` for persistent agent tools
